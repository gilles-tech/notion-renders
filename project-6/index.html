1 Arrive (6 tickets)
A01
Missing flow
P1
Demo path for visitors without a CV
"I want to see what this does before uploading my personal document"
A secondary CTA 'Watch a demo in 60 seconds' shows the Score Theatre on a fictional profile. Visitor sees the full journey without uploading anything. Reduces bounce from visitors who are curious but not committed.
A02
Missing interaction
P2
Dynamic social proof counter on landing
"Is this tool used by real people or is it abandoned?"
'2,847 CV analyzed this week' — must be live from database, not hardcoded. If count < 100, show institutional credibility instead: 'Used by CSE, outplacement firms, and career consultants.'
A03
Edge case
P2
Language detection: browser English but user is French
"Why is this page in the wrong language?"
Default to French (primary market). If browser lang is EN and no partner slug, show a discreet 'Switch to English' pill in the topbar. Never show English by default to a French visitor.
A04
Missing state
P1
Loading state after clicking 'Analyser mon CV'
"Did it work? Is it broken? Should I click again?"
After file selection: progress bar with 'Uploading...' then 'Analyzing...' with subtle pulse animation. If >5 seconds, show reassuring text: 'We are reading every detail of your CV.' Never a blank screen.
A05
Missing interaction
P2
Partner logos bar below the fold
"Who else trusts this tool?"
Row of partner category logos: 'CSE · Cabinets d outplacement · Organismes de prevoyance · Universites.' Even without signed logos, category labels add credibility.
A06
Edge case
P3
Accessibility: screen readers, contrast ratios, keyboard nav
"I have a visual impairment and I cannot use this tool"
WCAG 2.1 AA minimum. All interactive elements keyboard-focusable. Alt text on hero animation. Contrast ratio 4.5:1 on all text. Skip to main content link. ARIA labels on CTAs.
2 Discover (6 tickets)
D01
Missing state
P0
CV upload fails — error state
"Something went wrong and I do not know if it is my fault"
If upload fails: friendly message 'We could not read your file. Try a PDF or Word document under 10MB.' With a retry button. If parsing fails: 'We detected your CV but could not extract all sections. Would you like to proceed with partial analysis?' Never a generic error.
D02
Edge case
P2
Word doc with tables/columns that break parsing
"My CV has a creative layout and the tool cannot read it"
If parser confidence < 70%: show a warning 'Your CV uses a complex layout. For best results, try a single-column PDF version.' Offer a template download link. Never silently produce garbage results.
D03
Edge case
P2
Junior profile (<5 years) — tone and expectations
"I am early career, this tool seems designed for senior people"
Adaptive tone only covers 20+ years. Add a junior variant: simpler language, focus on potential not track record. Adapt sub-score expectations — Manager Pass weight is lower for juniors. Show 'entry-level' job row in NextJobs.
D04
Missing interaction
P1
State persistence — close browser mid-onboarding and return
"I closed my browser and now I have to start over"
Save onboarding state after each step. On return: 'Welcome back! You were at step 2. Resume where you left off?' with the last micro-result visible. Never restart from zero.
D05
Missing interaction
P2
Skip option for senior profiles who want results fast
"I do not want to answer questions, just show me my score"
After CV upload + first micro-result: 'Skip to full results' link (small, secondary). Senior execs value their time. The micro-result already proved value; let them proceed if they want.
D06
Edge case
P1
Gibberish input in open fields (the 'maison a larmor' problem)
"I misunderstood the question and typed something random"
Replace ALL open-ended questions with structured multi-choice (6 options + 'Autre'). If 'Autre' is selected, validate input length (min 3 words, max 50). Show placeholder example to guide the answer.
3 Orient (5 tickets)
O01
Inconsistency
P1
Timeline shows 4 steps in Hub but 7 stations existed — which is it?
"Confused about how many stages my transition has"
Resolve: Hub shows 4 PHASES (Resume, Jobs, Applications, Interviews). Each phase contains missions. The 7-station model maps onto these 4 phases. Document the mapping explicitly for the dev team.
O02
Missing state
P2
Hub on Day 30 / Day 60 — only Day 1 and Day 2 designed
"I have been using this for a month and the Hub still feels the same"
Design 4 Hub states: Day 1 (orientation), Day 2-7 (gamification appears), Day 8-30 (pipeline view: X applications, Y interviews), Day 30+ (negotiation mode: salary benchmarks, talking points).
O03
Missing screen
P2
Notification center — where do weekly reports and whispers live?
"I saw a notification but cannot find it again"
Small bell icon in topbar. Dropdown shows: weekly reports, whisper history, system messages (new job matches, score changes). Badge count for unread. Simple list, no inbox complexity.
O04
Missing screen
P1
The 'Coach' tab — what IS it?
"I tapped Coach and nothing happened"
Define Coach: either (a) AI chatbot for career questions, (b) resource library (articles, templates, checklists), or (c) connection to human advisor. Pick ONE for V1. If not ready, show 'Coming in Phase 2' with a preview of what it will be. Never an empty tab.
O05
Missing state
P2
Score is 90+ — first action 'fix your CV' does not apply
"The tool says fix my CV but my CV is already excellent"
Adaptive mission waterfall: if all sub-scores >75, skip CV improvement. First action becomes 'Explore your top 5 matching jobs' or 'Your profile is strong — start applying.' Celebrate the high score.

U04

P2
Before/after radar overlay on re-score — no mockup exists
"I improved my CV and want to see the visual difference"
Design the dual radar: old score in 30% opacity gray, new score in brand color. Each axis that improved shows green '+X'. Each regression shows red '-X'. Animation: new radar draws from center outward over 1.5 seconds.

P3

"My advisor asked for a report and I need to share something professional"
Design the 1-page PDF: score hero + radar + top 3 fixes + action plan. Branded with CareerNEXT logo + partner logo. Footer: 'Generated by CareerNEXT for [Name] on [Date].' Shareable with advisor.
U06

"My CV is great! Now what?"

5 Improve (6 tickets)

Missing screen
P0

"I want to pay but I do not know how much it costs or what I get"
Design the pricing page: Free tier (score + 2 sub-scores + 1 fix) vs Premium (all 7 + all fixes + ghost CV + PDF + benchmarking). Show monthly price, CSE activation code field, and 'Start free trial' option. Transparent, no surprises.
I02
Missing flow
P1
CSE activation code entry — where in the flow?
"My CSE gave me a code but I cannot find where to enter it"
Two entry points: (1) on pricing page, 'Have an activation code?' field, (2) on first freemium gate encounter, the blur overlay includes 'Enter your CSE code' alongside 'Start free trial.' Code validates instantly, unlocks premium.
I03
Missing interaction
P1
After payment — what is the first thing she sees?
"I just paid, I want to feel the value immediately"
Unlock animation: blurred CV preview de-blurs with a satisfying sweep (left to right, 1 second). All 7 sub-scores fade in simultaneously. Sound: improvement chime (opt-in). Text: 'Full analysis unlocked. Let us start with your biggest lever.' Auto-scrolls to Lever 1.
I04
Missing interaction
P2
Ghost CV overlay — the actual comparison interaction
"I uploaded a new version and want to see what changed"
Design the overlay: left column = old CV sections, right = new. Changed sections highlighted. Per-dimension delta badges: 'ATS: 71 → 84 (+13)' in green. Option: opacity slider to blend old/new. Dismiss returns to current CV view.
I05
Missing interaction
P2
AI-suggested rewrite UI — modal, inline, or panel?
"The tool suggested a rewrite and I want to accept it"
Inline: below the original bullet, a glass card shows the AI suggestion with 'Accept' and 'Edit' buttons. Accept replaces the text in a preview (not the actual CV — she still re-uploads). Edit opens an inline text field. Never a modal that blocks the page.
I06
Edge case
P3
CV is in English — language detection and adaptation
"My CV is in English but the interface is French"
Detect CV language from parser. If English: show sub-score explanations with English examples. Quick Fix keywords from English-language job market. Allow interface to stay French while CV analysis adapts to the document language.

Edge cases (5 tickets)
X01
Missing flow
P1
Pure B2C visitor — no CSE, found on Google. What changes?
"I am paying out of my own pocket, not through an employer"
No partner banner. Landing shows standard hero. Pricing page is the conversion path (not CSE code). Trial period (7 days free?). Different value prop: 'Invest in your career for the price of a dinner.'
X02
Edge case
P1
Entire journey on mobile (375px) — Score Theatre, Netflix rows, Studio
"I only have my phone and the experience should be the same quality"
Audit every screen at 375px. Score Theatre must work without phone frame. Netflix rows must scroll smoothly with touch. Studio detail panel must be full-screen on mobile. Bottom nav must be thumb-reachable. Test on iOS Safari + Chrome Android.
X03
Edge case
P3
RGPD right to be forgotten — where is the delete button?
"I want all my data removed permanently"
Account settings: 'Delete my account and all data' button. Confirmation modal: 'This will permanently delete your CV, score history, applications, and all personal data. This cannot be undone.' 30-day grace period before hard delete.
X04
Missing screen
P2
CSE Advisor experience — Advisor Mirror is barely specified
"I am an advisor tracking 30 candidates and need an overview"
Design the advisor dashboard: candidate list (name, score, trend arrow, last login, applications count), click to drill-down (read-only score page), assign missions, export CSV. Filter by date range, score range, activity status.
X05
Missing flow
P3
Candidate succeeds and signs — offboarding flow
"I signed a contract! What happens to my account?"
Celebration screen with confetti. 'Congratulations! You did it.' Option: generate Transition Certificate, share on LinkedIn, keep account active for future transitions (dormant mode), or delete data. The positive ending matters.
Business model (6 tickets)
B01
Conversion
P0
Pricing page — NEVER designed
"I want to pay but there is no clear pricing anywhere"
Design the pricing page with 2-3 tiers. Free: score + 2 sub-scores + 1 fix + blurred preview. Premium: everything + ghost CV + PDF + Career DJ + one-click apply. Enterprise/CSE: bulk pricing + Advisor Mirror. Clean, Stripe-style layout.
B02
Conversion
P1
Trial vs paid vs CSE-provided — how does she know her tier?
"I do not know what I am paying for or if my CSE covers this"
Account badge in topbar: 'Free' / 'Premium' / 'CSE — Renault.' Settings page shows: current plan, what is included, upgrade/downgrade options. If CSE-provided: 'Offered by your employer through [CSE name].'
B03
Missing screen
P2
Account settings — profile, password, notifications, data export
"I want to change my email or turn off notifications"
Standard settings page: profile (name, email, password), notification preferences (Career DJ email, weekly report, push), data export (download all my data as JSON/CSV), account tier, delete account. Essential for RGPD compliance.
B04
Missing flow
P2
Re-engagement email sequence — she signed up but does not return
"I forgot about this tool after the first visit"
Email sequence: Day 1 (welcome + score summary), Day 3 ('Your score could be 88 — here is how'), Day 7 ('3 new jobs match your profile'), Day 14 ('Your streak is at risk'). Each with a single CTA back to the relevant screen.
B05
Missing interaction
P3
Referral mechanism — 'Recommend to a colleague'
"This tool helped me, I want to share it with someone in the same situation"
Share button on success screens (score improvement, application sent, contract signed). Generates a personal referral link. 'Invite a colleague — they get 7 days free, you get 1 month extended.' Simple viral loop.
B06
Missing screen
P3
Help / FAQ / Support — no support channel specified
"Something is not working and I do not know who to contact"
Help icon in topbar or settings. FAQ page covering: 'How is my score calculated?', 'Is my data safe?', 'How do I delete my account?', 'My CV was not parsed correctly.' Contact form or chat for issues beyond FAQ. Response SLA shown.
